“commit” Registros hechos en git git diff Nos muestra las modificaciones que existen entre nuestro “commit” actual “Working Directory” y el que está almacenado en el “Repository” git checkout Actualiza los “commit” del “Working Directory” para que coincidan con la versión almacenada en el “Repository” (mantiene los “commit” guardados) git reset Resetea los “commit” del “Working Directory” para que coincidan con la versión almacenada en el “Repository” (elimina los “commit” guardados) git log muestra el historial del repositorio git. Los “commit” y un identificador único (hash SHA-1) git revert sirve para deshacer cambios efectuados en el historial de “commit” de un “Repository”. git commit --amend permite añadir o eliminar cambios preparados al “commit” más reciente. Puedes cambiar el nombre del “commit” agregando -m y el nombre nuevo. git mv Mover o renombrar un archivo git rm Eliminar archivos
